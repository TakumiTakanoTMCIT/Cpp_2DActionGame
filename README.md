# 横スクロール2Dアクションゲーム

SDL2 と C++ で制作している 2D アクションゲームです。
プレイヤーの移動、ジャンプ、敵との接触判定、踏みつけ、マグマ、ステージ遷移、タイトル画面、クリア画面、効果音までをひと通り実装しています。

「まずはゲームサイクルを最後まで成立させる」を目標に作っていて、現在は

- タイトル画面
- プレイ
- 死亡してリトライ
- ゴールして次ステージへ
- 最終ステージクリア

まで一連の流れが遊べる状態です。

# ポイント

このプロジェクトでは「ゲームとして最後まで遊べること」と「拡張しやすい設計」を意識しました。\
タイトル → プレイ → クリアまで一貫したゲームループを実装
当たり判定・ジャンプ・敵AIなど基本要素を実装
ステージを文字列から生成し、拡張しやすい構成に
クラスごとに責務を分離（App / Player / Stage / SoundManager）

# 技術的な工夫
- UpdateとDrawを分離し、処理の見通しと拡張性を向上
- main関数の肥大化を防ぐため設計を分割
- データ（文字列）からステージ生成する仕組みを採用

# 苦労した点
- 当たり判定とジャンプ挙動の調整（めり込み防止）
- 処理の整理（UpdateとDrawの分離）

## Features

- 左右移動とジャンプ
- ブロックとの当たり判定
- 敵との接触で死亡
- 敵を上から踏んで撃破
- マグマによる即死判定
- 敵の巡回移動
- ステージ文字列からマップを生成
- 複数ステージ対応
- タイトル画面 / クリア画面
- UI 決定音の再生

## Controls

- `Left Arrow`: 左移動
- `Right Arrow`: 右移動
- `Space`: ジャンプ / タイトル開始 / クリア画面終了
- `Enter`: タイトル開始 / クリア画面終了

## ファイル構成

```text
.
├── main.cpp
├── App.h / App.cpp
├── Player.h / Player.cpp
├── Stage.h / Stage.cpp
├── SoundManager.h / SoundManager.cpp
└── assets/
```

### ファイルの担当と説明

- `main.cpp`
  ゲーム全体の起動とメインループだけを担当します。

- `App.h / App.cpp`
  SDL の初期化、入力処理、更新、描画、終了処理など、アプリ全体の進行を管理します。

- `Player.h / Player.cpp`
  プレイヤーの移動、ジャンプ、重力、当たり判定、死亡判定、ゴール判定を担当します。

- `Stage.h / Stage.cpp`
  ステージデータ、ブロック、敵、マグマ、ゴール、ステージ生成を担当します。

- `SoundManager.h / SoundManager.cpp`
  UI 効果音の初期化、再生、終了処理を担当します。

## ビルド方法

Homebrew で必要なライブラリを導入したあと、以下のコマンドでビルドできます。

```bash
g++ main.cpp App.cpp SoundManager.cpp Stage.cpp Player.cpp -std=c++17 -I/opt/homebrew/include -L/opt/homebrew/lib -lSDL2 -lSDL2_image -lSDL2_ttf -lSDL2_mixer -o main
```

## 実行方法

```bash
./main
```

## ステージ

ステージは文字列配列から生成しています。
タイル文字ごとの意味は以下の通りです。

- `P`: プレイヤー開始位置
- `G`: 地面
- `B`: ブロック
- `E`: 敵
- `M`: マグマ
- `C`: ゴール
- `.`: 空白

この方式にしているので、マップの追加や調整をコード上で比較的簡単に行えます。
